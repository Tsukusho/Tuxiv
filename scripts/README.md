# マイグレーションスクリプト

このディレクトリには、データベースの一括変更を行うマイグレーションスクリプトが含まれています。

## 役職データの正規化 (migrate-normalize-roles.ts)

### 目的

全角カンマ（、）を含む役職データを半角カンマ（,）に統一し、配列として正しく分割されるように修正します。

### 背景

ユーザーが役職入力時に全角カンマ「、」を使用すると、以下のような問題が発生していました:

- **入力例**: `記録、役者、音響`
- **従来の動作**: `["記録、役者、音響"]` (1つの役職として保存)
- **期待される動作**: `["記録", "役者", "音響"]` (3つの役職として保存)

このマイグレーションスクリプトは、既存のデータベース内の全ての役職データを修正します。

### 実行方法

```bash
# 方法1: npmスクリプトを使用（推奨）
npm run migrate:roles

# 方法2: ts-nodeで直接実行
npx ts-node scripts/migrate-normalize-roles.ts
```

### 実行前の準備

1. **データベースのバックアップを取る**（重要！）
   ```bash
   # MongoDBのバックアップ例
   mongodump --uri="your-mongodb-uri" --out=./backup/$(date +%Y%m%d_%H%M%S)
   ```

2. **環境変数を確認**
   - `.env.local` ファイルに `MONGODB_URI` と `JWT_SECRET` が設定されていることを確認

### 処理内容

スクリプトは以下の処理を行います:

1. データベース内の全 `Availability` ドキュメントを取得
2. 各ドキュメントの `roles` フィールドをチェック
3. 全角カンマ（、）を含む役職を検出
4. 全角カンマを半角カンマに変換し、配列として分割
5. 重複を削除
6. データベースを更新

- `scripts/migration-roles-result.json`: 変更された全ドキュメントの詳細

### ロールバック方法

万が一問題が発生した場合、事前に取得したバックアップから復元できます:

```bash
# MongoDBのリストア例
mongorestore --uri="your-mongodb-uri" --drop ./backup/20250121_143000
```

### 注意事項

- ✅ **本番環境で実行する前に、必ず開発環境でテストしてください**
- ✅ **実行前に必ずデータベースのバックアップを取得してください**
- ⚠️ データ量が多い場合、処理に時間がかかる可能性があります
- ⚠️ 実行中はアプリケーションを停止することを推奨します

### トラブルシューティング

#### エラー: "データベースに接続できません"

- `.env.local` の `MONGODB_URI` が正しく設定されているか確認
- MongoDBサーバーが起動しているか確認

#### エラー: "ts-node: command not found"

```bash
npm install -g ts-node
# または
npx ts-node scripts/migrate-normalize-roles.ts
```
