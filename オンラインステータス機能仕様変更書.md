# オンラインステータス機能 仕様変更書

## 概要

日程調整機能の Availability モデルに「オンライン」ステータスを追加し、参加者がオンラインでの参加も可能であることを明示できるようにする機能追加です。

## マイグレーション手順

### 1. データベースマイグレーション

#### マイグレーションスクリプト作成

**ファイル**: `online_status_migration.js`

```javascript
// MongoDB migration script to add 'online' status to availability types
const { MongoClient } = require("mongodb");

const MONGODB_URI =
  process.env.MONGODB_URI || "mongodb://localhost:27017/tuxiv";

async function migrateOnlineStatus() {
  const client = new MongoClient(MONGODB_URI);

  try {
    await client.connect();
    console.log("MongoDB接続成功");

    const db = client.db();
    const collection = db.collection("availabilities");

    // 既存のavailabilityドキュメント数を確認
    const totalDocs = await collection.countDocuments();
    console.log(`総availability数: ${totalDocs}`);

    // 現在のtypeフィールドの分布を確認
    const typeDistribution = await collection
      .aggregate([
        { $unwind: "$availableSlots" },
        { $group: { _id: "$availableSlots.type", count: { $sum: 1 } } },
      ])
      .toArray();

    console.log("現在のtype分布:", typeDistribution);

    // スキーマバリデーションを一時的に無効化
    await db.runCommand({ collMod: "availabilities", validator: {} });
    console.log("スキーマバリデーションを一時的に無効化しました");

    // 新しいスキーマバリデーションを設定（onlineを追加）
    await db.runCommand({
      collMod: "availabilities",
      validator: {
        $jsonSchema: {
          bsonType: "object",
          required: [
            "eventId",
            "userId",
            "name",
            "grade",
            "roles",
            "availableSlots",
          ],
          properties: {
            eventId: { bsonType: "objectId" },
            userId: { bsonType: "objectId" },
            name: { bsonType: "string" },
            grade: { bsonType: "string" },
            roles: {
              bsonType: "array",
              items: { bsonType: "string" },
            },
            availableSlots: {
              bsonType: "array",
              items: {
                bsonType: "object",
                required: ["start", "end", "type"],
                properties: {
                  start: { bsonType: "date" },
                  end: { bsonType: "date" },
                  type: {
                    bsonType: "string",
                    enum: ["available", "undecided", "online"], // onlineを追加
                  },
                },
              },
            },
          },
        },
      },
    });

    console.log(
      "新しいスキーマバリデーションを設定しました（onlineステータス対応）"
    );

    // 既存データの整合性チェック
    const invalidDocs = await collection
      .find({
        availableSlots: {
          $elemMatch: {
            type: { $nin: ["available", "undecided", "online"] },
          },
        },
      })
      .toArray();

    if (invalidDocs.length > 0) {
      console.log(
        `警告: ${invalidDocs.length}件のドキュメントに無効なtype値が含まれています`
      );
      console.log(
        "無効なドキュメントのID:",
        invalidDocs.map((doc) => doc._id)
      );
    } else {
      console.log("全てのドキュメントが新しいスキーマに適合しています");
    }

    console.log("オンラインステータスマイグレーション完了");
  } catch (error) {
    console.error("マイグレーションエラー:", error);
    throw error;
  } finally {
    await client.close();
    console.log("MongoDB接続を閉じました");
  }
}

if (require.main === module) {
  migrateOnlineStatus()
    .then(() => {
      console.log("マイグレーションスクリプト実行完了");
      process.exit(0);
    })
    .catch((error) => {
      console.error("マイグレーションスクリプト実行エラー:", error);
      process.exit(1);
    });
}

module.exports = { migrateOnlineStatus };
```

## 影響範囲

### データベース

- **スキーマ変更**: `availabilities`コレクションの`availableSlots.type`フィールドの enum 値追加
- **既存データ**: 既存のデータは影響を受けない（後方互換性あり）
- **インデックス**: 既存のインデックスは影響を受けない

### バックエンド

- **API**: 既存の API エンドポイントは変更なし（リクエスト・レスポンス形式は同じ）
- **バリデーション**: 新しい値のバリデーション追加
- **エラーハンドリング**: 無効な type 値に対するエラーメッセージ追加

### フロントエンド

- **UI**: モード切り替えボタンの動作変更（2 択 →3 択）
- **表示**: カレンダーイベントの色分けに新しい色を追加
- **結果表示**: 参加状況の表示にオンライン参加者を追加

### 既存機能への影響

- **後方互換性**: 既存の'available'と'undecided'ステータスは変更なし
- **データ移行**: 既存データの移行は不要
- **API 互換性**: 既存の API クライアントは影響を受けない
