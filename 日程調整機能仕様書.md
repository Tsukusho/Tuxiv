# 日程調整機能 詳細仕様書

## 概要

Tuxiv アプリケーションの日程調整機能は、サークル活動などの予定調整を効率的に行うための機能です。参加者が自分の空き時間を入力し、管理者が最適な日程を決定できるシステムです。

## データベース仕様

### 1. ScheduleEvent コレクション

**ファイル**: `src/models/scheduleEvent.ts`

#### スキーマ定義

```typescript
interface IScheduleEvent extends Document {

  description?: string; // イベント説明（任意）

```

#### データベース構造

- **コレクション名**: `scheduleevents`
- **インデックス**: `createdBy` (作成者検索用)
- **タイムスタンプ**: `createdAt`, `updatedAt` (自動生成)

### 2. Availability コレクション

**ファイル**: `src/models/availability.ts`

#### スキーマ定義

```typescript
interface IAvailability extends Document {
  eventId: Schema.Types.ObjectId; // 関連するイベントID（必須）
  userId: Schema.Types.ObjectId; // ユーザーID（必須）
  name: string; // 参加者名（必須）
  grade: string; // 学年・代（必須）
  roles: string[]; // 役職の配列
  availableSlots: {
    // 空き時間スロットの配列
    start: Date; // 開始時刻（必須）
    end: Date; // 終了時刻（必須）
    type: "available" | "undecided"; // ステータス（必須、デフォルト: 'available'）
  }[];
}
```

#### データベース構造

- **コレクション名**: `availabilities`
- **インデックス**:
  - `eventId` (イベント別検索用)
  - `userId` (ユーザー別検索用)
  - `eventId + userId` (複合検索用)
- **タイムスタンプ**: `createdAt`, `updatedAt` (自動生成)

## バックエンド API 仕様

### 1. イベント作成 API

**エンドポイント**: `POST /api/schedule/events`

#### リクエスト

```typescript
{
  title: string;                    // イベントタイトル
  description?: string;             // イベント説明
  candidateDates: {                 // 候補日時
    start: string;                  // ISO形式の開始日時
    end: string;                    // ISO形式の終了日時
  }[]
}
```

#### レスポンス

```typescript
{
  _id: string;                      // 作成されたイベントID
  title: string;
  description?: string;
  candidateDates: { start: Date; end: Date; }[];
  createdBy: string;                // 作成者ID
  createdAt: Date;
  updatedAt: Date;
}
```

#### 認証

- JWT トークンによる認証が必要
- トークンは Cookie から取得

### 2. イベント詳細取得 API

**エンドポイント**: `GET /api/schedule/events/[id]`

#### レスポンス

```typescript
{
  event: {
    _id: string;
    title: string;
    description?: string;
    candidateDates: { start: string; end: string; }[];
    createdBy: string;
  };
  availabilities: {                 // 全参加者の出欠情報
    _id: string;
    name: string;
    grade: string;
    roles: string[];
    availableSlots: {
      start: string;
      end: string;
      type: 'available' | 'undecided';
    }[];
  }[];
  currentUser: {
    fullName: string;               // ログイン中のユーザー名
  };
  currentUserAvailability?: {       // ログイン中のユーザーの出欠情報（存在する場合）
    name: string;
    grade: string;
    roles: string[];
    availableSlots: {
      start: string;
      end: string;
      type: 'available' | 'undecided';
    }[];
  };
}
```

### 3. 出欠情報登録・更新 API

**エンドポイント**: `POST /api/schedule/events/[id]/availabilities`

#### リクエスト

```typescript
{
  name: string;                     // 参加者名
  grade: string;                    // 学年・代
  roles: string[];                  // 役職の配列
  availableSlots: {                 // 空き時間スロット
    start: string;                  // ISO形式の開始時刻
    end: string;                    // ISO形式の終了時刻
    type: 'available' | 'undecided'; // ステータス
  }[]
}
```

#### レスポンス

```typescript
{
  _id: string;                      // 更新された出欠情報のID
  eventId: string;
  userId: string;
  name: string;
  grade: string;
  roles: string[];
  availableSlots: {
    start: Date;
    end: Date;
    type: 'available' | 'undecided';
  }[];
  createdAt: Date;
  updatedAt: Date;
}
```

#### 処理ロジック

- **Upsert 処理**: 既存の出欠情報があれば更新、なければ新規作成
- **バリデーション**: 各スロットに`type`フィールドが存在しない場合は`'available'`をデフォルト値として設定

### 3. 結果表示画面

**ファイル**: `src/components/schedule/ResultsDashboard.tsx`

#### 機能

- **フィルター機能**:

  - 参加可能人数の最小値設定
  - 役職による絞り込み

  - ホバー時の詳細情報表示

#### 表示ロジック

```typescript
// 各時間スロットの参加状況計算
const allSlotEvents = slots.map((slot) => {
  const availableMembers: string[] = [];
  const undecidedMembers: string[] = [];

  // 各メンバーの空き時間をチェック
  for (const member of targetMembers) {
    const foundSlot = member.availableSlots.find((avail) => {
      const availStart = new Date(avail.start);
      const availEnd = new Date(avail.end);
      return availStart < slot.end && availEnd > slot.start;
    });

    if (foundSlot?.type === "undecided") {
      undecidedMembers.push(member.name);
    } else if (foundSlot) {
      availableMembers.push(member.name);
    }
  }

  // 参加率に基づく色計算
  const percentage =
    totalTargetMembers > 0 ? availableMembers.length / totalTargetMembers : 0;
  const green = Math.min(255, Math.round(180 * percentage));
  const color = `rgb(80, ${green + 75}, 150)`;

  return {
    start: slot.start,
    end: slot.end,
    title: `空:${availableMembers.length} 未:${undecidedMembers.length}`,
    backgroundColor: color,
    borderColor: color,
    extendedProps: {
      availableNames: availableMembers,
      undecidedNames: undecidedMembers,
      unavailableNames: unavailableMembers,
    },
  };
});
```

## 既知の制限事項

1. **単一イベント制限**: 現在は 1 つのイベント ID にハードコーディングされている
2. **リアルタイム更新**: 出欠情報の変更は他のユーザーにリアルタイムで反映されない
3. **通知機能**: イベント作成や出欠情報の変更通知は実装されていない
4. **権限管理**: イベント作成者以外の編集権限管理は未実装
