## 2. 課題と要件（作者のみ、タグだけ編集可能）

- 投稿後の編集は「タグのみ」。タイトル・説明・画像・NSFW・匿名は不可。
- 編集権限は「作者本人のみ」。匿名投稿でも作者本人は編集可。
- 監査性の担保: `updatedAt`は自動更新。必要なら編集履歴を別途検討（今回は対象外）。

## 5. 変更後コード案（実装案・参考。編集は行わない）

### 5.1 API: `PATCH /api/artworks/[id]/tags`

- 目的: 作者のみタグを更新
- 認可: JWT の`id`と`Artwork.userId`一致
- 入力: `{ tags: string[] }`（空配列不可、件数上限任意）
- 出力: 更新後の Artwork

```ts
// File: src/app/api/artworks/[id]/tags/route.ts（新設）
import { NextResponse } from "next/server";
import dbConnect from "@/lib/dbConnect";
import Artwork from "@/models/artwork";
import jwt from "jsonwebtoken";
import { cookies } from "next/headers";

export async function PATCH(
  req: Request,
  context: { params: Promise<{ id: string }> }
) {
  await dbConnect();
  const { id } = await context.params;

  const cookieStore = await cookies();
  const token = cookieStore.get("token")?.value;
  if (!token)
    return NextResponse.json({ error: "認証が必要です。" }, { status: 401 });

  let userId: string;
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET!) as {
      id: string;
    };
    userId = decoded.id;
  } catch {
    return NextResponse.json({ error: "認証が無効です。" }, { status: 401 });
  }

  const body = await req.json();
  const tags: unknown = body?.tags;
  if (
    !Array.isArray(tags) ||
    tags.length === 0 ||
    !tags.every((t) => typeof t === "string")
  ) {
    return NextResponse.json(
      { error: "tagsは非空の文字列配列で指定してください。" },
      { status: 400 }
    );
  }

  const artwork = await Artwork.findById(id);
  if (!artwork)
    return NextResponse.json(
      { error: "作品が見つかりません。" },
      { status: 404 }
    );
  if (artwork.userId.toString() !== userId) {
    return NextResponse.json(
      { error: "編集権限がありません。" },
      { status: 403 }
    );
  }

  artwork.tags = tags.map((t) => t.trim()).filter(Boolean);
  await artwork.save();
  return NextResponse.json(artwork, { status: 200 });
}
```

```tsx
// File: src/components/TagEditor.tsx（新設・提案）
"use client";
import { useState } from "react";
export default function TagEditor({
  artworkId,
  initialTags,
}: {
  artworkId: string;
  initialTags: string[];
}) {
  const [input, setInput] = useState(initialTags.join(", "));
  const [saving, setSaving] = useState(false);
  const onSave = async () => {
    setSaving(true);
    const tags = input
      .split(",")
      .map((t) => t.trim())
      .filter(Boolean);
    const res = await fetch(`/api/artworks/${artworkId}/tags`, {
      method: "PATCH",
      headers: { "Content-Type": "application/json" },
      credentials: "include",
      body: JSON.stringify({ tags }),
    });
    setSaving(false);
    if (res.ok) location.reload();
    else alert("タグの更新に失敗しました");
  };
  return (
    <div className="space-y-2">
      <input
        className="input-field w-full"
        value={input}
        onChange={(e) => setInput(e.target.value)}
      />
      <button className="btn-primary" onClick={onSave} disabled={saving}>
        {saving ? "保存中..." : "保存"}
      </button>
    </div>
  );
}
```

## 6. バリデーション・セキュリティ指針

- タグ最大数・各タグ最大長（例: 30 文字）・総文字数上限（例: 300 文字）を API 側で実施推奨。
- タグの正規化（前後空白除去・重複排除・小文字化など）方針を明記。
- 作者認可はサーバー側で必ず判定。クライアント条件分岐は「表示制御」のみ。

---

## 7. テスト観点

- 作者本人が PATCH 成功する。
- 他人が PATCH すると 403。
- 未ログインは 401。
- 無効入力（空配列、非文字列、過大件数、過長タグ）で 400。
- 更新後、`GET /api/artworks/[id]`で新しいタグが返る。
- 詳細ページでタグ表示が更新される。

---

## 8. マイグレーションの必要性

- スキーマ変更なし（既存`tags: string[]`を更新するのみ）。
- 既存データ互換性に影響なし。

---

## 9. 影響範囲

- 新規 API 追加のみで既存 API は不変。
- UI は所有者にのみ追加コンポーネントを表示。
- 検索、タイムライン、サムネ一覧には副作用なし。

---

## 10. リスクとフォールバック

- リスク: 不正なタグ入力・XSS（タグそのものは検索キーワード用途、画面表示時はサニタイズ済みリンク文字列）。
- フォールバック: 保存失敗時はエラートースト、リトライ導線。
